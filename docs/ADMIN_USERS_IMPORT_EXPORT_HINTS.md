# Подсказка: импорт и экспорт пользователей (админка)

Код не описывается здесь построчно — только то, что нужно знать при заполнении Excel и при чтении экспорта.

## Организация (колонка «Организация»)

В базе и при проверке импорта допускаются **только такие строки** (как в выпадающем списке в форме пользователя, но **без** префикса «Система»):

| Значение в Excel | Комментарий |
|------------------|-------------|
| `ДИТ` | ДИТ |
| `112` | **Не** «Система 112» — только код `112` |
| `101` | **Не** «Система 101» — только код `101` |
| `Танто-С` | Танто-С |

Если указать, например, `Система 112`, импорт вернёт ошибку **«некорректная организация»** — так задано в валидации на сервере.

## Уровень доступа (колонка «Уровень доступа»)

Допустимые значения:

| Значение | Смысл |
|----------|--------|
| `read` | Только чтение |
| `edit` | Чтение и редактирование |

## Суперпользователь и активность

Колонки **«Суперпользователь»** и **«Активен»** (если есть): для импорта распознаются как «да» значения вроде `1`, `true`, `да`, `yes`, `y` (без учёта регистра). Пустая **«Активен»** считается как активный пользователь.

## Импорт: обязательные колонки (первая строка — заголовки)

Имена сравниваются **без учёта регистра** и с обрезкой пробелов. Обязательно должны быть (по смыслу):

- ФИО  
- Организация  
- Почта  
- Пароль  
- Уровень доступа  

Дополнительно можно использовать альтернативные заголовки там, где это поддержано импортом: `Email`, `Password`, `Access level` и т.п. (см. код обработчика импорта, если нужны точные синонимы).

Пароль: **не короче 6 символов**. Импорт **только добавляет** новых пользователей; если почта уже есть в системе — строка с ошибкой.

## Гранты карточки (колонка «Гранты карточки (CSV)»)

Несколько ключей через **запятую**, без пробелов или с пробелами — лишнее обрежется. В импорте на сервере разрешены **только** такие ключи (остальные в строке игнорируются):

| Ключ | Назначение (кратко) |
|------|---------------------|
| `comment` | Комментарии к предложению |
| `shortName` | Краткое наименование |
| `initiator` | Инициатор |
| `responsible` | Ответственный (в UI формы может называться иначе — в импорте для файла используется именно **`responsible`**) |
| `sectionName` | Раздел |
| `proposalText` | Текст предложения |
| `problemComment` | Комментарии и описание проблем |
| `discussionSummary` | Обсуждение |
| `implementationQueue` | Приоритет (очередь) |
| `noteText` | Примечание |
| `completedAt` | Дата выполнения |
| `ditOutgoing` | Письмо в ДИТ |
| `attachments` | Вложения |
| `deleteRequirement` | Удаление предложений (см. ниже) |

**Важно:** для пользователей с уровнем `read` ключ `deleteRequirement` из CSV **не применяется** (его сбрасывают при импорте). Удаление предложений по смыслу завязано на уровень `edit` и отдельные правила в приложении.

## Гранты ГК (колонка «Гранты ГК (CSV)»)

Через запятую, разрешённые ключи:

| Ключ | Смысл |
|------|--------|
| `gkContractEdit` | Редактирование договора ГК |
| `gkStageEdit` | Редактирование этапов |
| `gkFunctionEdit` | Редактирование функций / вложений импорта ГК (по правилам приложения) |

Для пользователя **без** суперпользователя и с уровнем **не** `edit` гранты ГК из файла **не сохраняются** (очищаются при импорте).

## Экспорт

В выгрузке те же колонки, что и в шаблоне/ожидании импорта по смыслу: ФИО, организация, почта, уровень, флаги, CSV грантов, дата создания. **Пароля в экспорте нет** — для новых строк в импорте пароль нужно задать вручную в таблице.

## Быстрая памятка по ошибке «некорректная организация»

Замените подпись из интерфейса на **код**: «Система 112» → **`112`**, «Система 101» → **`101`**.
